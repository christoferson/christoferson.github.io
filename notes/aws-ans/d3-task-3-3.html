Knowledge 1: Connectivity Options and Their Use Cases

  • VPC Peering vs Transit Gateway (Network Architecture Comparison):
    • VPC Peering (Direct VPC-to-VPC Connection):
      • Architecture Characteristics:
        • Direct One-to-One Connectivity:
          • Purpose: Direct network path between two VPCs
          • Implementation:
            • Dedicated network connection
            • No intermediate hops
            • Direct routing path
          • Benefits:
            • Simplified connectivity
            • Reduced complexity
            • Direct path control
        • Performance Characteristics:
          • Lower Latency:
            • Direct path routing: No intermediate processing
            • Same-region benefits: AWS backbone utilization
            • Performance metrics:
              • Sub-millisecond latency in same AZ
              • Single-digit ms in same region
              • Predictable performance
        • Cost Considerations:
          • No Bandwidth Charges Between AZs:
            • Same-AZ traffic: Free data transfer
            • Cross-AZ traffic: Standard AZ data transfer rates
            • Cost benefits:
              • Reduced data transfer costs
              • Predictable pricing
              • No additional service charges
        • Routing Characteristics:
          • Non-transitive Routing:
            • Connection Behavior: Direct peer only
            • Implications:
              • No indirect connectivity
              • Each connection is isolated
              • Explicit peering required
            • Design Considerations:
              • Connection planning
              • Route table management
              • Scaling implications
      • Use Cases:
        • Simple Network Requirements:
          • Two VPC connectivity
          • Direct resource access
          • Low latency requirements
        • Development and Testing:
          • Dev/Test environment connection
          • Isolated testing scenarios
          • Limited scope deployments
    • Transit Gateway (Centralized Network Hub):
      • Architecture Characteristics:
        • Hub-and-Spoke Topology:
          • Design Pattern: Centralized connectivity model
          • Components:
            • Central hub: Transit Gateway
            • Spoke attachments: VPCs, VPNs, DX
            • Route tables: Traffic control
          • Benefits:
            • Simplified architecture
            • Reduced connection count
            • Centralized control
        • Management Features:
          • Centralized Management:
            • Single Control Point:
              • Route management
              • Policy enforcement
              • Monitoring and visibility
            • Operational Benefits:
              • Simplified operations
              • Consistent policy application
              • Enhanced visibility
        • Routing Capabilities:
          • Transitive Routing:
            • Routing Behavior: Full mesh capability
            • Features:
              • Route propagation
              • Path selection
              • Route table segmentation
        • Account Integration:
          • Multi-Account Support:
            • Organization Integration:
              • Resource sharing
              • Cross-account attachments
              • Centralized billing
      • Use Cases:
        • Enterprise Networks:
          • Large-scale deployments
          • Multiple VPC connectivity
          • Hybrid cloud integration
        • Complex Routing Requirements:
          • Multi-region connectivity
          • Advanced routing policies
          • Segmented network zones

Implementation Best Practices:

  • Architecture Selection (Design Decision Framework):
    • Solution Assessment:
      • Selection Criteria:
        • Choose VPC Peering When:
          • Simple point-to-point connectivity needed
          • Low latency is critical: Direct connection requirement
          • Cost optimization priority: No additional gateway charges
          • Limited number of connections: Less than 10 VPCs
        • Choose Transit Gateway When:
          • Complex network topology required
          • Multiple VPC connections needed: More than 10 VPCs
          • Centralized control required: Single management point
          • Multi-account or multi-region deployment
    • Design Considerations:
      • Network Planning:
        • CIDR Planning:
          • Non-overlapping ranges: Prevent routing conflicts

          • Subnet strategy: Consistent sizing across VPCs

          • Path selection strategy: Traffic engineering
  • Security Implementation (Access Control Strategy):

      • Network Segmentation:
