AWS Certified Advanced Networking - Specialty (ANS-C01)

Direct Connect

AWS Direct Connect - Core Concepts

What is AWS Direct Connect?

  • Definition and Purpose:
    • Dedicated network connection from on-premises to AWS
    • Bypasses internet service providers in your network path
    • Uses standard Ethernet fiber-optic cable
    • Provides consistent network performance with reduced bandwidth costs
  • Connection Types:
    • Dedicated Connections: 1, 10, 100, or 400 Gbps ports dedicated to a single customer
    • Hosted Connections: 50 Mbps to 10 Gbps connections provided by AWS Direct Connect Partners
    • All connections require single-mode fiber with specific transceiver types

Virtual Interface Types

  • Private Virtual Interface:
    • Connects to Amazon VPC using private IP addresses
    • Requires virtual private gateway or Direct Connect gateway
    • Supports jumbo frames up to 9001 bytes MTU
  • Public Virtual Interface:
    • Connects to all AWS public services using public IP addresses
    • Cannot use Elastic IPs or BYOIP addresses from Amazon pool
    • Limited to 1500 bytes MTU
  • Transit Virtual Interface:
    • Connects to Transit Gateway for multi-VPC and multi-account access
    • Requires Direct Connect gateway
    • Supports jumbo frames up to 8500 bytes MTU

Direct Connect Gateway

  • Purpose and Function:
    • Globally available resource for connecting to multiple VPCs across regions
    • Acts as distributed BGP route reflectors
    • Operates outside the data traffic path
    • Enables access to VPCs in any AWS Region (except China)
  • Association Types:
    • Virtual Private Gateway associations (up to 20 VPCs)
    • Transit Gateway associations (up to 3 Transit Gateways)
    • Cannot mix association types on same Direct Connect gateway

AWS Direct Connect - Resiliency and High Availability

Resiliency Models

  • Maximum Resiliency (99.99% SLA):
    • Separate connections terminating on separate devices at multiple locations
    • Protects against device failures, connectivity failures, and complete location failures
    • Recommended for mission-critical enterprise workloads
    • Most comprehensive protection against all failure scenarios
  • High Resiliency (99.9% SLA):
    • Two single connections to multiple locations
    • Protects against fiber cuts, device failures, and location failures
    • Suitable for production workloads requiring high availability
    • Less complex than maximum resiliency model
  • Development and Test (No specific SLA):
    • Separate connections on separate devices at a single location
    • Protects against device failures only
    • Suitable for non-critical workloads and development environments
    • No protection against location failures

Failover Testing

  • AWS Direct Connect Failover Test:
    • Tests connection resiliency without physical disconnection
    • Disables BGP peering session between on-premises and AWS
    • Duration: 1-180 minutes (configurable)
    • Test history preserved for 365 days
  • Test Capabilities:
    • Works with all virtual interface types
    • Tests both IPv4 and IPv6 BGP sessions
    • Only virtual interface owner can initiate tests
    • Can be canceled while running

Link Aggregation Groups (LAGs)

  • Purpose and Function:
    • Logical interface using LACP (Link Aggregation Control Protocol)
    • Aggregates multiple connections at a single Direct Connect endpoint
    • Simplifies management of multiple connections
    • Operates in Active/Active mode only
  • Key Limitations:
    • MLAG (Multi-chassis LAG) is NOT supported
    • Maximum 4 connections for speeds < 100 Gbps
    • Maximum 2 connections for 100/400 Gbps
    • All connections must be same bandwidth and terminate at same endpoint
  • Operational Features:
    • Configurable minimum operational connections threshold
    • LAG becomes non-operational if connections fall below threshold
    • Single MACsec configuration applies to all member connections

AWS Direct Connect - Security and Encryption

Data Protection Options

  • Default Behavior:
    • Traffic is not encrypted by default
    • Requires explicit encryption configuration
    • Multiple encryption options available based on requirements
  • Encryption Methods:
    • AWS Site-to-Site VPN over Direct Connect
    • MAC Security (MACsec)
    • Transit Gateway with VPN attachments
    • Application-level encryption

MACsec (MAC Security)

  • Overview:
    • IEEE 802.1AE standard for Layer 2 encryption
    • Provides data confidentiality, integrity, and origin authenticity
    • Point-to-point encryption between customer router and AWS Direct Connect router
    • No additional charge for MACsec capability
  • Connection Support:
    • Available on 10, 100, and 400 Gbps dedicated connections only
    • Not available on 1 Gbps connections or hosted connections
    • Requires MACsec-capable interface on customer device
    • Connection must be Layer 2 transparent
  • Technical Requirements:
    • 10 Gbps: Supports GCM-AES-256 and GCM-AES-XPN-256
    • 100/400 Gbps: Requires GCM-AES-XPN-256 only
    • 256-bit keys only
    • SCI (Secure Channel Identifier) must be enabled
    • No support for dot1q-in-clear (VLAN tag outside encrypted payload)

Identity and Access Management

  • Authentication Methods:
    • IAM users, roles, and federated identities
    • BGP MD5 authentication (mandatory)
    • Service-linked roles for MACsec integration
  • Key Managed Policies:
    • AWSDirectConnectFullAccess
    • AWSDirectConnectReadOnlyAccess
    • AWSDirectConnectServiceRolePolicy (for service-linked role)
  • Resource-Level Permissions:
    • Support for resource ARNs (connections, LAGs, virtual interfaces, gateways)
    • Tag-based access control
    • Cross-account access capabilities

AWS Direct Connect - Routing and BGP Configuration

BGP Requirements and Configuration

  • Basic Requirements:
    • BGP (Border Gateway Protocol) required for all Direct Connect connections
    • BGP MD5 authentication mandatory
    • One BGP session per IP family per virtual interface
    • Maximum 100 routes per BGP session for private/transit VIFs
    • Maximum 1000 routes for public VIFs
  • ASN Configuration:
    • 16-bit ASN range: 64512-65534 (private)
    • 32-bit ASN range: 1-2147483647
    • Cannot use same ASN for customer and AWS gateways
    • Must use unique ASNs for multi-region Transit Gateways
  • BGP Timers:
    • Hold timer: 90 seconds default, 3 seconds minimum
    • Keepalive timer: 30 seconds default, 1 second minimum
    • Graceful restart timer: 120 seconds
    • BFD minimum interval: 300ms, multiplier: 3

Traffic Engineering with BGP

  • Path Selection Methods (Priority Order):
    • 1. Longest Prefix Match
    • 2. Local Preference
    • 3. AS_PATH Length
    • 4. MED (Multi-Exit Discriminator)
  • Local Preference BGP Communities:
    • 7224:7100 - Low Preference (backup/passive paths)
    • 7224:7200 - Medium Preference (default paths)
    • 7224:7300 - High Preference (primary/active paths)
    • Communities are mutually exclusive
  • Public VIF BGP Communities:
    • 7224:9100 - Local AWS Regions
    • 7224:9200 - Continental Regions (North America, Asia Pacific, EMEA)
    • 7224:9300 - Global (all public AWS Regions)
    • 7224:8100 - Same Region Routes
    • 7224:8200 - Same Continent Routes

Allowed Prefixes Behavior

  • Virtual Private Gateway Associations:
    • Acts as prefix filter only
    • Only advertises actual VPC CIDR
    • Must be same or wider than VPC CIDR
    • Example: For VPC CIDR 10.0.0.0/16, allowed prefix 10.0.0.0/15 will advertise 10.0.0.0/16
  • Transit Gateway Associations:
    • Directly advertises allowed prefixes
    • Independent of attached VPC CIDRs
    • Originates from Direct Connect gateway ASN
    • Example: For VPC CIDR 10.0.0.0/16, allowed prefix 22.0.0.0/24 will advertise 22.0.0.0/24
  • Important Limitations:
    • No prefix overlaps allowed with multiple transit gateways
    • Cannot use 0.0.0.0/0 with multiple transit gateways
    • Each prefix must be unique across associations

AWS Direct Connect - Gateway Integrations

Direct Connect Gateway with Virtual Private Gateway

  • Key Capabilities:
    • Connects DX connection to VPCs across accounts and regions
    • Uses private virtual interface
    • Enables multi-VPC connectivity through single connection
    • Global resource - create in any region
  • Critical Rules:
    • One VGW per Direct Connect gateway
    • VGW must be attached to a VPC
    • Cannot attach if already associated with Transit Gateway
    • No overlapping CIDR blocks between associated VPCs
    • Do not enable route propagation before gateway association
  • Association Process:
    • Association proposals expire after 7 days if not accepted
    • Accepted/deleted proposals visible for 3 days
    • VGW detachment from VPC auto-disassociates from DX gateway

Direct Connect Gateway with Transit Gateway

  • Key Capabilities:
    • Connect DX connection to multiple VPCs/VPNs through Transit Gateway
    • Uses transit virtual interface
    • Enables hub-and-spoke networking model
    • Maximum 3 Transit Gateway associations per DX gateway
  • Critical Rules:
    • Cannot attach if already associated with VGW
    • Cannot attach if connected to private virtual interface
    • Must use unique ASNs for multi-region Transit Gateways
    • Point-to-point /30 ranges don't propagate to Transit Gateway
    • Association can take up to 40 minutes to establish
  • Cross-Account Associations:
    • Transit Gateway owner creates association proposal
    • DX Gateway owner must accept proposal
    • Can include allowed prefix specifications
    • DX Gateway owner can override requested prefixes

Direct Connect Gateway with Cloud WAN

  • Key Capabilities:
    • Routes traffic between core network edge locations and DX connections
    • Uses shortest available path
    • Enables central policy-based management
    • Supports automated routing through BGP
  • Critical Limitations:
    • One core network per DX gateway
    • Single segment association only
    • Cannot associate with other AWS region resources simultaneously
    • No allowed prefixes list support
    • All segment prefixes advertised to DX gateway
    • ASN must be outside core network ASN range
  • Management Scope:
    • Created and managed from Cloud WAN Console
    • Uses Network Manager for administration
    • Direct Connect console shows read-only information

AWS Direct Connect - SiteLink and Advanced Features

AWS Direct Connect SiteLink

  • Purpose and Function:
    • Enables direct PoP-to-PoP connectivity between Direct Connect locations
    • Uses shortest available AWS network path
    • Bypasses regional routing requirements
    • Reduces latency between on-premises sites
  • Configuration Requirements:
    • Enable on VIFs at two or more Direct Connect locations
    • All locations must attach to same Direct Connect gateway
    • Works with existing Direct Connect connections
    • No new connections required
  • Support Matrix:
    • Supported: Transit virtual interfaces, Private VIF with Direct Connect gateway
    • Not Supported: Private VIF attached to virtual gateway, Public virtual interfaces
    • Not available in AWS GovCloud (US) or China Regions
  • Technical Specifications:
    • BGP required
    • IPv6 supported
    • MACsec supported (if port/PoP supports it)
    • No managed QoS functionality, but DSCP markings preserved
    • BGP Communities for path preference:
      • 7224:7100 - Low preference
      • 7224:7200 - Medium preference
      • 7224:7300 - High preference

AWS Local Zones Integration

  • Connectivity Options:
    • Direct Connect can connect to Local Zones
    • Data travels directly to/from Local Zones
    • Bypasses AWS Region
    • Improves performance and reduces latency
  • Configuration Process:
    • Extend VPC from parent Region
    • Create new subnet and assign to Local Zone
    • Private Connection: Associate VGW to Direct Connect private VIF or DX Gateway
    • Public Connection: Use Direct Connect Public VIF with IGW
  • Key Limitations:
    • Transit Gateway not supported in Local Zones
    • MTU size: 1468 (vs 9001 in Regions)*
    • Single flow limit: ~2.5 Gbps (vs 5 Gbps in Regions)
    • Ingress routing goes through parent Region first
    • *Exception: Los Angeles Local Zone not affected by MTU and flow limits

Remote Region Access

  • Global Access Capabilities:
    • Access to all public Regions from any DX location
    • Includes AWS GovCloud (US)
    • Excludes China Regions (Beijing and Ningxia)
    • Uses AWS global network backbone
  • Remote Public Service Access:
    • Requires public virtual interface
    • Established BGP session
    • Router learns routes for all public AWS Regions
    • Uses AWS IP Address Ranges (published in documentation)
  • Remote VPC Access:
    • Primary Method: Direct Connect Gateway
    • Alternative Method: VPN over Public VIF
    • Outbound data from remote Region charged at that Region's rate

AWS Direct Connect - Maintenance and Operations

Maintenance Events

  • Types of Maintenance:
    • Planned Maintenance:
      • Scheduled in advance
      • Improves availability
      • Delivers new features
    • Emergency Maintenance:
      • Critical service-impacting failures
      • Requires immediate AWS action
      • Not planned in advance
  • Notification Timeline:
    • Planned Maintenance:
      • 14 calendar days notice
      • 7 calendar days notice
      • 1 calendar day notice
      • Calendar days include non-business days and holidays
    • Emergency Maintenance:
      • Up to 60 minutes prior notification
      • Cannot be canceled
  • Notification Methods:
    • AWS Personal Health Dashboard
    • Email notifications
    • AWS Direct Connect Console subscriptions

Monitoring and Metrics

  • CloudWatch Key Metrics:
    • ConnectionState: 1 = up, 0 = down
    • ConnectionBpsEgress/Ingress: Bandwidth utilization
    • ConnectionPpsEgress/Ingress: Packet rates
    • ConnectionErrorCount: Includes CRC errors
    • ConnectionLightLevelTx/Rx: Fiber connection health (dBm)
    • VirtualInterfaceBpsEgress/Ingress: VIF bandwidth
  • Metric Collection:
    • Default interval: 5 minutes
    • Minimum interval: 1 minute
    • Error counts reset after each reporting interval
    • Light level normal range: -10 to -3 dBm
  • Monitoring Tools:
    • CloudWatch Alarms for threshold-based notifications
    • CloudTrail for API activity logging
    • Direct Connect Console for connection status
    • CloudWatch Dashboard for metrics visualization

Troubleshooting Common Issues

  • Connection Issues:
    • Light Levels: Below -10 dBm indicates potential problems
    • Auto-Negotiation: Must be disabled for speeds > 1 Gbps
    • Physical connectivity: Check cross-connect and fiber issues
  • BGP Session Problems:
    • ASN mismatch
    • MD5 authentication failure
    • Route limit exceeded (>100 for private/transit VIF, >1000 for public VIF)
    • BGP timer misconfigurations
  • Recovery Times:
    • BGP reconvergence: 3-5 minutes
    • LAG member failure: Under 1 minute
    • Full connection failure: Based on failover design
