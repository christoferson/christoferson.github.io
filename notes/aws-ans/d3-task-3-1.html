Knowledge 1: Industry-Standard Routing Protocols in Hybrid Networks

  • BGP over Direct Connect (Border Gateway Protocol Implementation):
    • Protocol Configuration (BGP Setup and Management):
      • AS Number Management: Autonomous System identification and control
        • ASN Types:
          • Private ASNs (64512-65534):
            • Purpose: Internal network identification
            • Usage: Non-internet facing connections
            • Best Practice: Use in private peering scenarios
          • Public ASNs (1-64495):
            • Purpose: Internet-facing BGP routing
            • Usage: When connecting to internet providers
            • Requirement: IANA registration needed
        • ASN Assignment Strategy:
          • AWS Side:
            • Virtual Private Gateway ASN
            • Transit Gateway ASN
            • Direct Connect Gateway ASN
          • Customer Side:
            • Router ASN configuration
            • ASN consistency across locations
            • ASN path planning
      • Route Advertisement: Network prefix announcement control
        • Prefix Management:
          • Route Filters:
            • Purpose: Control route propagation
            • Implementation:
              • Prefix lists: Specific network ranges
              • AS-path filters: Path-based control
              • Community filters: Tag-based filtering
          • Advertisement Control:
            • Selective announcement: Specific VPC routes
            • Aggregation: Summarized routes
            • Default routes: Internet access control
      • Path Selection: Route preference determination
        • BGP Decision Process:
          • Path Attributes:
            • Weight: Local router preference
            • Local Preference: AS-wide preference
            • AS Path Length: Shortest path selection
            • Origin: Route source type
            • MED: Multi-exit discriminator
          • Selection Criteria:
            • Primary path: Best path selection
            • Backup paths: Failover options
            • Load sharing: Equal cost paths
      • BGP Attributes: Route characteristic control
        • Well-Known Attributes:
          • AS_PATH:
            • Purpose: Path vector tracking
            • Manipulation: Path prepending
            • Loop prevention: AS path checking
          • NEXT_HOP:
            • Purpose: Next router identification
            • Modification: Next-hop self
            • Reachability: Next-hop validation
        • Optional Attributes:
          • Communities:
            • Purpose: Route tagging and policy
            • Types: Standard and extended
            • Usage: Traffic engineering

Implementation Best Practices:

  • BGP Design (Protocol Implementation):
    • Session Configuration:
      • Purpose: Ensure stable and secure BGP operations
      • Best Practices:
        • Authentication:
          • Always enable MD5 authentication: Prevent unauthorized peers
          • Implement key rotation schedule: Regular security updates
          • Document authentication details: Secure key management
        • Timer Settings:
          • Keepalive: 1/3 of hold time (typical 30s)
          • Hold time: Balance between detection and stability (typical 90s)
          • BFD when available: Sub-second failure detection
      • Anti-patterns:
        • Disabling authentication: Security risk
        • Too aggressive timers: Network instability
        • Missing monitoring: Delayed problem detection
    • Route Management:
      • Purpose: Maintain efficient and secure route distribution
      • Implementation:
        • Route Filtering:
          • Implement strict prefix filters: Control advertised routes
          • Use AS path filters: Prevent unauthorized announcements
          • Apply route aggregation: Reduce table size
        • Path Selection:
          • Use communities for policy: Traffic engineering
          • Implement consistent MED: Predictable routing
          • Plan AS path prepending: Traffic steering

AWS-Specific Considerations:

  • Service Limits and Quotas: Technical constraints
    • Direct Connect Limitations:
      • BGP Session Limits:
        • Maximum prefixes: 100 routes per VGW (default)
        • Virtual interfaces per connection: Up to 50
        • BGP ASN range: 1-2147483647 (public), 64512-65534 (private)
      • Performance Constraints:
        • Maximum MTU: 1500 or 9001 (jumbo frames)
        • BFD minimum interval: 300ms
        • Route convergence time: Typically 3-5 seconds
    • Transit Gateway Considerations:
      • Routing Capacity:
        • Routes per route table: 10,000 dynamic routes
        • BGP ASN support: Both 2-byte and 4-byte
        • Route propagation: Automatic between attachments
      • Attachment Limits:
        • Maximum attachments: 5000 per TGW
        • VPC attachments per region: Region-specific limits
        • DX Gateway attachments: 20 per TGW
  • Cost Implications: Financial considerations
    • Direct Connect Costs:
      • Port hours: Fixed monthly cost
      • Data transfer: Per GB charges
      • Virtual interface charges: Private/transit VIFs
    • Transit Gateway Costs:
      • Attachment hours
      • Data processing
      • Inter-region peering
  • Operational Requirements: Management considerations
    • Monitoring Setup:
      • CloudWatch Metrics:
        • Connection status
        • BGP status
        • Route counts
        • Bandwidth utilization
      • Logging Requirements:
        • VPC Flow Logs
        • CloudTrail for API activity
        • BGP session logs
