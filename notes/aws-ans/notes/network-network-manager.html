AWS Global Networks for Transit Gateways - Study Guide

Introduction to AWS Global Networks

  • Definition:
    • AWS Global Networks enables centralized management of networks across AWS accounts, Regions, and on-premises locations
    • Acts as a container for network objects including transit gateways and on-premises resources
    • Provides visualization and monitoring capabilities through Network Manager console
  • Key Components:
    • Global network - High-level container for network objects
    • Device - Physical or virtual appliance in on-premises network or cloud
    • Connection - Connectivity between two devices
    • Link - Single internet connection from a site
    • Site - Physical on-premises location (branch, office, store, campus, data center)

Home Region Concept

  • Definition:
    • Region where data related to global network is aggregated and stored
    • Currently only US West (Oregon) is supported as home Region
    • For GovCloud, data is aggregated to AWS GovCloud (US-West)
    • Once established, home Region cannot be changed
  • Data Aggregation:
    • Uses Amazon SQS and S3 for data transfer and storage
    • Includes topology data, event data, transit gateway IDs
    • Optional location, provider, and link data
    • All data transfer occurs over secure, encrypted channels

AWS PrivateLink Support

  • Key Features:
    • Creates private connectivity between Network Manager and VPCs
    • Eliminates need for internet gateway or NAT devices
    • Only supports IPv6 dual-stack endpoints
    • Available only in us-west-2 and us-gov-west-1 Regions

Exam Tips

  • Key Points to Remember:
    • Global Networks does not create, modify, or delete transit gateways
    • Home Region is fixed as US West (Oregon)
    • IPv6 is supported on dual-stack endpoints
    • Available in all major AWS Regions (30+ regions)
    • Uses Amazon Location Service for map creation

How AWS Global Networks for Transit Gateways Works

Global Network Implementation

  • Creation Process:
    • Start by creating an empty global network framework
    • Register existing transit gateways to the network
    • Define on-premises resources (sites, devices, links)
    • Visualize and monitor through Network Manager dashboard
  • Monitoring Capabilities:
    • View network activity and health through CloudWatch metrics
    • Track network events through CloudWatch Events
    • Identify issues across AWS and on-premises resources
    • Monitor connections between different network segments

Transit Gateway Registration

  • Automatic Inclusion of Attachments:
    • VPCs
    • Site-to-Site VPN connections
    • AWS Direct Connect gateways
    • Transit Gateway Connect
    • Transit gateway peering connections
  • Peering Considerations:
    • When registering a transit gateway with peering attachment, peer transit gateway is visible
    • Peer transit gateway's attachments are not visible unless also registered
    • Deleted transit gateways are automatically deregistered

Multi-Region and Multi-Account Support

  • Benefits:
    • Monitor global health of AWS network across regions and accounts
    • Unified view of distributed network resources
    • Centralized management of complex network topologies

On-Premises Network Integration

  • Resource Definition Process:
    • Add sites with location information (address, coordinates)
    • Define devices representing physical/virtual appliances
    • Create links representing internet connections
    • Associate devices with sites and links
  • AWS Connection Methods:
    • Associate customer gateway with device
    • Establish Connect peer from virtual appliance to transit gateway
    • Partner integration through AWS Partner Network (APN)

Supported Resource Types

  • Amazon VPC Resources:
    • Transit gateways and their attachments
    • Transit gateway route tables
    • Transit Gateway Connect peers
    • VPCs
  • AWS VPN Resources:
    • Customer gateways
    • VPN connections
  • AWS Direct Connect Resources:
    • Direct Connect connections
    • Direct Connect gateways
    • Virtual interfaces
  • Network Manager Resources:
    • Connections
    • Devices
    • Links
    • Sites

Getting Started with AWS Global Networks

Implementation Process

  • Prerequisites:
    • Existing transit gateway with attachments
    • Transit gateway can be in same account or different account within organization
  • Basic Setup Steps:
    • Create a global network container
    • Register transit gateways
    • Define on-premises resources (optional)
    • Enable multi-account access (optional)
    • View and monitor through dashboard

Multi-Account Access

  • Requirements:
    • AWS Organizations setup
    • Enabling trusted access for Network Manager
    • Registering delegated administrators
  • Benefits:
    • Register transit gateways from multiple accounts
    • View and manage resources across organization
    • Centralized monitoring of distributed networks

Common Scenarios

  • AWS-Only Multi-Region Network:
    • Transit gateways across multiple regions and accounts
    • Peering connections between transit gateways
    • No on-premises components
  • Single Device with VPN Connection:
    • One site with single device and link
    • Site-to-Site VPN connection to transit gateway
    • Transit gateway with VPC attachments
  • Multi-Device Configurations:
    • Device with multiple VPN connections
    • Multi-device and multi-link sites
    • SD-WAN connecting to AWS
    • Connections between devices

Exam Tips

  • Key Concepts to Remember:
    • Global network is just a container - requires additional configuration
    • Transit gateways can only be registered to one global network
    • On-premises resources must be manually defined
    • Multi-account requires AWS Organizations setup
    • Network Manager console provides visualization but not creation of transit gateways

Multi-Account Support in AWS Global Networks

Multi-Account Overview

  • Definition:
    • Ability to manage, monitor, and view global network resources from multiple AWS accounts
    • Requires AWS Organizations setup
    • Not available in AWS GovCloud Regions
  • Implementation Recommendation:
    • Use global networks console for enabling multi-account settings
    • Console automatically creates required roles and permissions
    • Alternative approaches require advanced expertise

Trusted Access

  • Definition:
    • Creates AWSServiceAccess for global networks and AWS CloudFormation StackSets
    • Provides permissions for AWS Organizations to deploy service-linked roles
  • Permission Levels:
    • Administrator - Can modify resources from other accounts
    • Read-only - Can only view information about resources
  • Deployed Roles:
    • Network Manager SLR to all member accounts
    • CloudFormation StackSets roles
    • CloudWatch sharing and monitoring roles
    • Global networks console switch role

Delegated Administrators

  • Capabilities:
    • Leverage service-linked roles across accounts
    • View transit gateways from other member accounts
    • Register transit gateways to global network
    • Same permissions as management account
  • Limitations:
    • Only member accounts can be registered as delegated administrators
    • Maximum of ten delegated administrators per organization
    • Requires trusted access to be enabled first
  • Management Account Consideration:
    • Not recommended to use management account for global network management
    • Required service-linked roles not propagated to management account

Exam Tips

  • Key Points to Remember:
    • Multi-account requires AWS Organizations setup
    • Console is recommended method for enabling multi-account
    • Maximum of ten delegated administrators allowed
    • Management account should not be used to manage global network
    • Disabling trusted access affects visibility of resources from other accounts

AWS Global Networks Monitoring - CloudWatch Metrics and Events

AWS Monitoring Tools Overview

  • Available Monitoring Tools:
    • Amazon CloudWatch: Real-time monitoring of resources and applications
    • Amazon EventBridge: Near-real-time stream of system events
    • AWS CloudTrail: Record of API calls and actions taken by users or services
  • Monitoring Capabilities:
    • Collect and track metrics
    • Create customized dashboards
    • Set alarms for threshold violations
    • Enable automated event-driven responses
    • Maintain audit trails of network changes

CloudWatch Metrics for Global Networks

  • Key Features:
    • Metrics retained for 15 months for historical analysis
    • Near-real-time visibility into network performance
    • Ability to set alarms and trigger notifications
  • Available Metric Categories:
    • Transit gateway metrics
    • Site-to-Site VPN connection metrics
    • On-premises resource metrics (devices and links)

On-Premises Resource Metrics

  • Device Metrics:
    • DataIn: Bytes received by the device
    • DataOut: Bytes sent by the device
    • TunnelDownCount: Number of VPN tunnels with DOWN status
  • Device Metric Dimensions:
    • DeviceId: Filters metric data by specific device
  • Link Metrics:
    • DataIn: Bytes received by on-premises network via link
    • DataOut: Bytes sent from on-premises network via link
  • Link Metric Dimensions:
    • LinkId: Filters metric data by specific link

Viewing CloudWatch Metrics

  • Available Views:
    • Global network metrics filtered by transit gateway
    • Specific transit gateway and its attachments
  • Console Navigation:
    • Network Manager console → Global networks → Transit gateway network → Monitoring
    • Network Manager console → Global networks → Transit gateways → [TGW ID]
  • Additional Features:
    • Custom time frame selection (Relative or Absolute)
    • UTC or local time zone display
    • Add to CloudWatch dashboard (US West Oregon Region only)

AWS Global Networks Monitoring - CloudTrail Integration

CloudTrail Overview for Global Networks

  • Key Capabilities:
    • Records all API calls for global network operations
    • Captures console actions and programmatic API calls
    • Provides audit trail for security and compliance
    • Enabled by default on AWS accounts
  • Information Captured:
    • What request was made to global networks
    • Source IP address of the request
    • Identity of the requester (user, role, service)
    • Timestamp of the request
    • Additional request details

CloudTrail Event Types

  • Event History:
    • Recent events viewable in CloudTrail console
    • Searchable and downloadable
    • Limited retention period
  • CloudTrail Trails:
    • Continuous delivery of events to S3 bucket
    • Can apply to all AWS Regions
    • Enables long-term storage and analysis
    • Integration with other AWS services for further analysis

Global Network Actions in CloudTrail

  • Logged Operations:
    • All global network API actions (e.g., CreateGlobalNetwork)
    • Console operations
    • Actions from AWS CLI or SDKs
  • Identity Information:
    • Root vs IAM user credentials
    • Temporary security credentials (roles, federated users)
    • Actions performed by other AWS services
    • Captured in userIdentity element of log entries

Exam Tips for CloudWatch and CloudTrail

  • CloudWatch Key Points:
    • 15-month retention for metrics
    • Different metric types for devices vs links
    • Ability to filter by specific resources using dimensions
    • Dashboard integration limited to US West (Oregon) Region
  • CloudTrail Key Points:
    • Enabled by default but trails require configuration
    • All global network API calls are logged
    • Identity information helps with security auditing
    • Trails can aggregate logs across regions
  • Use Cases:
    • Performance monitoring of network components
    • Security and compliance auditing
    • Troubleshooting connectivity issues
    • Historical analysis of network changes
    • Automated alerting for network issues

AWS Global Networks Monitoring with EventBridge - Study Guide

Introduction to Global Network Monitoring

  • Background:
    • AWS Global Networks for Transit Gateways provides centralized management of your network infrastructure
    • EventBridge delivers near-real-time stream of system events describing resource changes
    • Monitoring global networks is essential for maintaining network reliability and security
  • Key Benefits:
    • Near real-time visibility into network changes
    • Automated response to network events
    • Centralized monitoring of distributed network resources
    • Ability to track topology changes, routing updates, and status changes

Getting Started with EventBridge for Network Monitoring

  • Prerequisites:
    • Onboarding to CloudWatch Logs Insights is required
    • IAM permissions needed for onboarding:
      • events:PutTargets, events:DescribeRule, events:PutRule
      • logs:PutResourcePolicy, logs:DescribeLogGroups
      • logs:DescribeResourcePolicies, logs:CreateLogGroup
  • Onboarding Process:
    • CloudWatch event rule created in US West (Oregon) Region
    • CloudWatch Logs log group created in US West (Oregon) Region
    • Event rule configured with log group as target
    • CloudWatch resource policy created for trust relationship

Event Types Overview

  • Three Main Categories of Events:
    • Topology change events - track structural changes to your network
    • Routing update events - monitor changes to transit gateway route tables
    • Status update events - track connectivity status changes

AWS Global Networks Monitoring - Topology Change Events

Understanding Topology Change Events

  • Definition:
    • Events that occur when there are structural changes to resources in your global network
    • Provide visibility into network component additions, modifications, and removals
    • All events include resource ARNs, timestamps, and detailed change information
  • Common Event Structure:
    • Source: aws.networkmanager
    • Detail-type: Network Manager Topology Change
    • Resources: Global network ARN and affected resource ARN
    • Detail: Contains changeType, changeDescription, and resource-specific information

Key Topology Change Event Types

  • Transit Gateway Events:
    • TGW-DELETED: Transit gateway removed from global network
    • TGW-PEERING-CREATED: Peering connection established between transit gateways
    • TGW-PEERING-DELETED: Peering connection removed
  • VPN Connection Events:
    • VPN-CONNECTION-CREATED: New Site-to-Site VPN connection created
    • VPN-CONNECTION-DELETED: VPN connection removed
    • VPN-CONNECTION-CUSTOMER-GATEWAY-MODIFIED: Customer gateway changed
    • VPN-CONNECTION-TARGET-GATEWAY-MODIFIED: Target gateway changed
  • VPC Attachment Events:
    • VPC-ATTACHMENT-CREATED: VPC attached to transit gateway
    • VPC-ATTACHMENT-DELETED: VPC detached from transit gateway
  • Direct Connect Events:
    • DXGW-ATTACHMENT-CREATED: Direct Connect gateway attached
    • DXGW-ATTACHMENT-DELETED: Direct Connect gateway detached
  • Connect Attachment Events:
    • CONNECT_ATTACHMENT_CREATED: Connect attachment created
    • CONNECT_ATTACHMENT_DELETED: Connect attachment deleted
    • TGW-CONNECT-PEER-CREATED: Connect peer created in attachment
    • CONNECT_PEER_DELETED: Connect peer deleted from attachment

Exam Tips for Topology Change Events

  • Key Information to Remember:
    • All topology events use "Network Manager Topology Change" as detail-type
    • Events contain both the global network ARN and specific resource ARN
    • Each event includes region information where the change occurred
    • Events provide detailed descriptions of what changed and how
  • Use Cases:
    • Tracking network expansion and contraction
    • Auditing network configuration changes
    • Triggering automated responses to network topology changes
    • Creating historical records of network evolution

AWS Global Networks Monitoring - Routing Update Events

Understanding Routing Update Events

  • Definition:
    • Events that occur when there are changes to transit gateway route tables
    • Provide visibility into route additions and removals
    • Critical for understanding network path changes and troubleshooting connectivity
  • Common Event Structure:
    • Source: aws.networkmanager
    • Detail-type: Network Manager Routing Update
    • Resources: Global network ARN and transit gateway ARN
    • Detail: Contains changeType, transitGatewayRouteTableArns, and route information

Key Routing Update Event Types

  • Route Table Events:
    • TGW-ROUTE-INSTALLED: Routes added to transit gateway route table
    • TGW-ROUTE-UNINSTALLED: Routes removed from transit gateway route table
    • CONNECT_PEER_DELETED: Transit gateway attachment's route table changed
  • Route Information Details:
    • destinationCidrBlock: Network prefix for the route
    • attachments: List of attachments associated with the route
    • routeType: Type of route (static, propagated)
    • routeState: State of the route (active, blackhole)
    • sequenceNumber: Ordering identifier for route updates

Exam Tips for Routing Update Events

  • Key Information to Remember:
    • Routing events use "Network Manager Routing Update" as detail-type
    • Events include sequence numbers to track order of updates
    • Route information includes CIDR blocks, attachment details, and route states
    • Multiple route changes may be included in a single event
  • Use Cases:
    • Monitoring route propagation across your network
    • Detecting unexpected route changes
    • Troubleshooting connectivity issues
    • Validating route policy enforcement

AWS Global Networks Monitoring - Status Update Events

Understanding Status Update Events

  • Definition:
    • Events that occur when there are changes to connectivity status
    • Focus primarily on VPN connections and BGP session status
    • Critical for real-time monitoring of network health
  • Common Event Structure:
    • Source: aws.networkmanager
    • Detail-type: Network Manager Status Update
    • Resources: Global network ARN and affected resource ARN
    • Detail: Contains changeType, changeDescription, and connection-specific information

Key Status Update Event Types

  • VPN Connection Status Events:
    • VPN-CONNECTION-IPSEC-DOWN: IPsec tunnel went down
    • VPN-CONNECTION-IPSEC-UP: IPsec tunnel came back up
    • VPN-CONNECTION-BGP-DOWN: BGP session went down
    • VPN-CONNECTION-BGP-ESTABLISH: BGP session established
  • Connect Peer Status Events:
    • CONNECT_PEER_BGP_DOWN: GRE tunnel BGP session went down
    • CONNECT_PEER_BGP_UP: GRE tunnel BGP session came back up
  • Status Event Details:
    • Region/edgeLocation: Where the status change occurred
    • outsideIpAddress: External IP address for VPN connections
    • peerAsn: Autonomous System Number of the peer
    • Relevant ARNs: For transit gateway, VPN connection, or Connect peer

Exam Tips for Status Update Events

  • Key Information to Remember:
    • Status events use "Network Manager Status Update" as detail-type
    • Events provide specific information about which component failed
    • Both IPsec (encryption) and BGP (routing) status are tracked separately
    • Connect peer events track GRE tunnel BGP status
  • Use Cases:
    • Real-time monitoring of network connectivity
    • Triggering automated failover mechanisms
    • Alerting operations teams to connectivity issues
    • Correlating application issues with network status
    • Tracking historical uptime and stability metrics
