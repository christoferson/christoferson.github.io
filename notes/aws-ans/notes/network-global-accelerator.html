AWS Global Accelerator - Secure VPC Connections

Introduction

  • Core Functionality:
    • Enables direct internet traffic flow to private subnet endpoints
    • Works with Network Load Balancers, internal Application Load Balancers, and EC2 instances
    • Provides secure access without requiring public IP addresses

Key Requirements

  • VPC Configuration:
    • Internet gateway must be attached to the VPC
    • No internet gateway route required for the subnet
    • No public IP addresses needed on resources
  • Comparison to Standard Internet Gateway:
    • Standard internet access requires:
      • Public IP addresses
      • Internet gateway routes
    • Global Accelerator overrides these requirements

Traffic Flow Mechanics

  • Routing Behavior:
    • Global Accelerator overrides typical internet routes
    • All logical connections arriving through Global Accelerator return through Global Accelerator
    • Traffic bypasses standard internet gateway path
  • Security Considerations:
    • Security groups apply to all traffic
    • Private subnets recommended for controlled access
    • Public subnets possible but not typical

Client IP Address Preservation

  • Requirements:
    • Special configuration needed for client IP preservation
    • Specific endpoint types support this feature
    • Additional security group considerations apply

Exam Tips

  • Key Points:
    • Internet gateway required at VPC level
    • No public IPs or routes needed at subnet level
    • Global Accelerator creates a secure path to private resources
    • Security groups still apply to all traffic
  • Common Scenarios:
    • Securing access to internal applications
    • Exposing private resources globally
    • Simplifying network security architecture
    • Controlling internet access to VPC resources

AWS Global Accelerator - Resilience Architecture

Introduction

  • Resilience Foundation:
    • Built on AWS global infrastructure
    • Leverages AWS Regions and Availability Zones
    • Designed for high availability and fault tolerance

Network Zone Architecture

  • Network Zone Concept:
    • Similar to an AWS Availability Zone
    • Isolated unit with dedicated physical infrastructure
    • Independent failure domain
  • IP Address Distribution:
    • IPv4 accelerator: Two static IPv4 addresses
    • Dual-stack accelerator: Four static IP addresses (2 IPv4 + 2 IPv6)
    • Each address served from a different network zone
    • Each IP address from a unique IP subnet per address family

Resilience Mechanisms

  • Network Zone Failover:
    • If one IP address becomes unavailable:
      • Due to IP address blocking
      • Due to network disruptions
    • Client applications can retry on healthy IP from other zone
    • Automatic isolation of failures
  • Health Monitoring:
    • Continuous endpoint health monitoring
    • Instant traffic redirection from unhealthy endpoints
    • Automatic failover to healthy endpoints

Integration with AWS Services

  • ARC Integration:
    • Respects Amazon Application Recovery Controller (ARC) traffic redirection
    • Supports zonal shift and zonal autoshift
    • Reroutes traffic from impaired Availability Zones
  • VPC Block Public Access:
    • Respects both bidirectional and ingress-only modes
    • Blocks traffic to VPCs with feature enabled
    • Honors exclusion configurations

Exam Tips

  • Key Resilience Features:
    • Two network zones for IPv4 accelerators
    • Four IP addresses (2 IPv4, 2 IPv6) for dual-stack accelerators
    • Automatic health-based routing
    • Integration with ARC for zonal recovery
  • Common Scenarios:
    • Multi-region application deployment
    • High-availability architecture design
    • Disaster recovery planning
    • Network disruption handling
  • Remember:
    • Network zones are independent failure domains
    • Client applications must implement retry logic
    • Health checks determine endpoint availability
    • Global Accelerator respects other AWS security services

AWS Global Accelerator - Infrastructure Security

Introduction

  • Security Foundation:
    • Protected by AWS global network security
    • Managed service with built-in protections
    • Follows AWS security best practices

Protocol and Encryption Requirements

  • Transport Layer Security (TLS):
    • TLS 1.2 required minimum
    • TLS 1.3 recommended
    • Ensures secure API communication
  • Cipher Suite Requirements:
    • Must support Perfect Forward Secrecy (PFS)
    • Requires DHE (Ephemeral Diffie-Hellman) or
    • ECDHE (Elliptic Curve Ephemeral Diffie-Hellman)
    • Most modern systems (Java 7+) support these modes

Authentication and Authorization

  • Request Signing:
    • All requests must be signed
    • Requires access key ID
    • Requires secret access key
    • Must be associated with IAM principal
  • Temporary Credentials:
    • AWS Security Token Service (STS) supported
    • Generates temporary security credentials
    • Used to sign requests
    • Enhances security through limited-time access

Security Design Principles

  • Infrastructure Protection:
    • Follows AWS Well-Architected Framework
    • Implements security pillar best practices
    • Provides defense-in-depth approach
  • DDoS Protection:
    • Static IP addresses protected by AWS Shield
    • Edge network provides distributed defense
    • Absorbs volumetric attacks

Exam Tips

  • Key Security Requirements:
    • TLS 1.2+ for API communication
    • PFS cipher suites (DHE/ECDHE)
    • Signed requests with IAM credentials
    • Optional STS temporary credentials
  • Security Integration:
    • Works with AWS Shield for DDoS protection
    • Respects VPC security controls
    • Integrates with IAM for access control
  • Remember:
    • Global Accelerator is a managed service
    • Security is shared responsibility
    • API security requirements are strict
    • Modern cipher suites are required

AWS Global Accelerator - Service Overview and Components

What is AWS Global Accelerator?

  • Core Service Definition:
    • AWS service for creating accelerators
    • Improves application performance for local and global users
    • Routes traffic over the AWS global network
    • Provides static IP addresses as entry points
  • Accelerator Types:
    • Standard Accelerator:
      • Improves availability for global audiences
      • Directs traffic to nearest Region endpoints
      • Optimizes for performance and availability
    • Custom Routing Accelerator:
      • Maps users to specific destinations
      • Deterministic routing to specific endpoints
      • Ideal for gaming, VoIP, and similar applications

Key Components

  • Static IP Addresses:
    • IPv4 accelerator: Two static IPv4 addresses
    • Dual-stack accelerator: Four addresses (2 IPv4 + 2 IPv6)
    • Anycast from AWS edge network
    • Remain assigned for accelerator lifetime
    • Optional BYOIP support for IPv4
  • Accelerator:
    • Core resource that directs traffic
    • Contains one or more listeners
    • Standard or custom routing type
    • IPv4 or dual-stack address type
  • DNS Name:
    • Default format: a1234567890abcdef.awsglobalaccelerator.com
    • Dual-stack format: a1234567890abcdef.dualstack.awsglobalaccelerator.com
    • Points to static IP addresses
    • Can be used with custom domain names
  • Network Zone:
    • Isolated unit with dedicated infrastructure
    • Similar to AWS Availability Zone concept
    • Each static IP served from different zone
    • Provides fault isolation
  • Listener:
    • Processes inbound connections
    • Configured for ports and protocols
    • Supports TCP, UDP, or both
    • Associated with endpoint groups
  • Endpoint Group:
    • Associated with specific AWS Region
    • Contains one or more endpoints
    • Can adjust traffic percentage with traffic dial
    • Enables performance testing across Regions
  • Endpoint:
    • Standard accelerator endpoints:
      • Network Load Balancers
      • Application Load Balancers
      • EC2 instances
      • Elastic IP addresses
    • Custom routing accelerator endpoints:
      • VPC subnets with EC2 instances

Service Integration

  • AWS Service Integrations:
    • Amazon Application Recovery Controller (ARC)
    • VPC Block Public Access
    • AWS Shield for DDoS protection
    • IAM for access control

Exam Tips

  • Key Concepts to Remember:
    • Static IP addresses persist until accelerator deletion
    • Network zones provide fault isolation
    • Standard vs. custom routing accelerator differences
    • Component hierarchy: Accelerator → Listener → Endpoint Group → Endpoint
  • Important Limitations:
    • Custom routing accelerators support IPv4 only
    • Dual-stack requires compatible endpoints
    • BYOIP available for IPv4 addresses only
    • Deleting accelerator loses static IP addresses permanently

AWS Global Accelerator - How It Works

Traffic Flow Overview

  • Basic Operation:
    • Static IP addresses serve as fixed entry points
    • Traffic enters AWS global network at closest edge location
    • Routed over optimized AWS backbone network
    • Directed to appropriate regional endpoints
  • Routing Logic by Accelerator Type:
    • Standard Accelerator:
      • Routes to optimal endpoint based on multiple factors
      • Considers user location, endpoint health, configured weights
      • Optimizes for performance and availability
    • Custom Routing Accelerator:
      • Routes each client to specific EC2 instance and port
      • Based on external IP address and listener port mapping
      • Deterministic routing with no automatic failover

Performance Optimization

  • TCP Connection Handling:
    • Terminates TCP connections at AWS edge locations
    • Establishes new TCP connection with endpoints
    • Results in faster response times (lower latency)
    • Increases throughput for applications
  • Path Optimization:
    • Maximizes time traffic spends on AWS network
    • Routes over well-monitored, congestion-free paths
    • Uses redundant AWS global network
    • Ensures optimal network path selection

Client IP Address Handling

  • IP Preservation Options:
    • Custom Routing Accelerator: Always preserves client IP
    • Standard Accelerator: Optional preservation for some endpoint types
    • Depends on endpoint type and configuration
  • Requirements:
    • Specific endpoint types support preservation
    • May require additional configuration
    • Affects security group settings

Health Monitoring and Failover

  • Standard Accelerator:
    • Continuous endpoint health monitoring
    • Instant redirection of new connections from unhealthy endpoints
    • Enables high-availability architecture
    • Existing connections maintained until completion or timeout
  • Custom Routing Accelerator:
    • No health checks used
    • No automatic failover
    • Destination specified by application logic
    • Application must handle failover scenarios

Traffic Management Features

  • Traffic Dials:
    • Control percentage of traffic to endpoint groups
    • Dial up/down traffic to specific Regions
    • Useful for performance testing or stack upgrades
    • Available for standard accelerators only
  • Endpoint Weights:
    • Fine-grained control within endpoint groups
    • Specify proportion of traffic to each endpoint
    • Values from 0-255 (default: 128)
    • Available for standard accelerators only

Special Considerations

  • Endpoint Weight Overrides:
    • In limited scenarios, Global Accelerator may override weights
    • Prioritizes availability over exact weight adherence
    • Helps avoid connection collisions
    • Common with client IP preservation
  • IP Fragmentation Handling:
    • UDP fragments forwarded to configured endpoint
    • TCP fragments dropped at edge (not supported)
    • TCP MSS negotiated automatically
    • Large packets (1500+ bytes) may require fragmentation

Exam Tips

  • Key Operational Concepts:
    • Traffic enters at edge, travels over AWS backbone
    • Standard accelerator optimizes for performance and health
    • Custom routing accelerator provides deterministic mapping
    • TCP connections terminated at edge for performance
  • Remember:
    • Health checks only apply to standard accelerators
    • Client IP preservation varies by accelerator type
    • Traffic management features (dials/weights) for standard accelerators only
    • TCP fragments dropped, UDP fragments supported

AWS Global Accelerator - Types and Idle Timeout

Accelerator Types in Detail

  • Standard Accelerator:
    • Optimizes for availability and performance
    • Routes based on:
      • Geographic proximity (lowest latency)
      • Endpoint health status
      • Configured weights and traffic dials
    • Ideal for:
      • General web applications
      • APIs and microservices
      • Multi-region deployments
      • High-availability architectures
    • Supports traffic management features:
      • Blue/green deployments
      • A/B testing
      • Traffic shifting across regions
  • Custom Routing Accelerator:
    • Provides deterministic routing to specific destinations
    • Maps users to specific EC2 instances and ports
    • No automatic health-based routing
    • Ideal for:
      • VoIP applications
      • Online gaming (player session mapping)
      • Real-time applications requiring specific instance routing
    • Limitations:
      • IPv4 support only (no dual-stack)
      • VPC subnet endpoints only
      • No automatic failover

Idle Timeout Behavior

  • Timeout Settings:
    • TCP connections: 340 seconds (fixed)
    • UDP connections: 30 seconds (fixed)
    • Not customizable
  • Connection Management:
    • Global Accelerator closes connection after idle timeout
    • Requires at least 1 byte of data within timeout window
    • Data can be in either direction (ingress or egress)
    • TCP keep-alive packets insufficient to maintain connection
  • Traffic Routing Behavior:
    • Continues routing established connections to endpoint until:
      • Idle timeout is reached
      • Connection naturally terminates
    • Even if endpoint is marked unhealthy or removed
    • New endpoint selection only occurs:
      • When new connection starts
      • After idle timeout occurs

Exam Tips

  • Key Differences Between Types:
    • Standard: Performance optimization, automatic health routing
    • Custom: Deterministic mapping, application-controlled routing
    • Standard: Supports all endpoint types
    • Custom: VPC subnets only
  • Timeout Considerations:
    • Fixed values: 340s for TCP, 30s for UDP
    • Cannot be modified
    • TCP keep-alive packets don't prevent timeout
    • Requires actual data transmission
  • Common Scenarios:
    • Choosing accelerator type based on application needs
    • Understanding connection persistence during endpoint changes
    • Planning for timeout-related connection behavior
    • Implementing proper keep-alive strategies

AWS Global Accelerator - Static IP Addresses

Static IP Address Overview

  • Default Allocation:
    • IPv4 accelerator: Two static IPv4 addresses
    • Dual-stack accelerator: Four addresses (2 IPv4 + 2 IPv6)
    • Anycast from AWS edge network
    • Automatically provisioned during accelerator creation
  • IP Address Sources:
    • Default: Amazon pool of IP addresses
    • Optional: Bring Your Own IP (BYOIP) for IPv4 addresses
    • BYOIP available for standard accelerators only

Benefits of Static IP Addresses

  • Simplified Access Management:
    • Single fixed entry point for clients
    • Consistent access regardless of backend changes
    • One-time allowlisting in client applications
    • One-time firewall rule configuration
    • One-time DNS record creation
  • Operational Advantages:
    • Easier regional expansion
    • Simplified application migration between regions
    • Consistent client connectivity during changes
    • Distributed traffic across multiple endpoints

IP Address Persistence

  • Lifecycle Management:
    • Static IPs remain assigned for accelerator lifetime
    • Persist even when accelerator is disabled
    • Lost permanently when accelerator is deleted
    • Cannot be reclaimed after deletion
  • Protection Mechanisms:
    • IAM policies can restrict accelerator deletion
    • Tag-based permissions recommended
    • Prevents accidental IP address loss

IPv6 Address Considerations

  • Dual-Stack Configuration:
    • IPv6 addresses allocated from same two /64 CIDR prefixes
    • Simplifies allowlisting and ACL controls
    • Requires dual-stack compatible endpoints
  • Endpoint Requirements:
    • IPv4-only endpoints work with IPv4 accelerators
    • Dual-stack accelerators require dual-stack compatible endpoints
    • See documentation for supported endpoint types

DNS Integration

  • Default DNS Names:
    • Standard format: a1234567890abcdef.awsglobalaccelerator.com
    • Dual-stack format: a1234567890abcdef.dualstack.awsglobalaccelerator.com
    • Points to all static IP addresses
  • Custom Domain Integration:
    • Can associate custom domain with static IPs
    • Requires additional DNS configuration
    • Supports both direct IP and CNAME approaches

Exam Tips

  • Critical Points:
    • Static IPs persist until accelerator deletion
    • Deletion permanently loses IP addresses
    • BYOIP available for IPv4 only
    • Dual-stack requires compatible endpoints
  • Best Practices:
    • Use IAM policies to prevent accidental deletion
    • Document static IP addresses
    • Consider IP persistence in disaster recovery planning
    • Understand endpoint compatibility requirements
  • Common Scenarios:
    • Multi-region application deployment
    • Regional migration planning
    • Custom domain integration
    • IP address management
