Accelerated AWS Site-to-Site VPN Connections

This section covers accelerated Site-to-Site VPN connections, an AWS feature that uses AWS Global Accelerator to improve the performance and reliability of your VPN connections. Learn how to configure and manage accelerated VPN connections.

    Overview

  • Accelerated Site-to-Site VPN connections use AWS Global Accelerator to route traffic through AWS's congestion-free global network.
  • The goal is to optimize network paths and improve application performance by routing traffic to the closest AWS edge location to the customer gateway device.
  • By enabling acceleration, you avoid potential network disruptions that could occur when traffic is routed over the public internet.

    Creating Accelerated VPN Connections

  • When creating a Site-to-Site VPN connection, acceleration is disabled by default.
  • You can enable acceleration when creating a new Site-to-Site VPN attachment on a transit gateway.
  • Acceleration uses a separate pool of IP addresses for the tunnel endpoints, selected from two distinct network zones.

    Key Features of Accelerated VPN Connections

  • Two accelerators are created and managed by AWS—one for each VPN tunnel.
  • It’s important to note that the accelerators cannot be managed through the AWS Global Accelerator console or APIs.
  • Accelerated VPN connections can only be used with AWS transit gateways, not virtual private gateways.
  • These connections cannot be used with AWS Direct Connect public virtual interfaces.

    Enabling Acceleration

  • Acceleration can only be enabled when creating a new Site-to-Site VPN connection. It cannot be applied to an existing VPN connection.
  • When creating the new VPN connection, configure the customer gateway device to use it and then delete the old VPN connection if necessary.
  • NAT-traversal (NAT-T) is required for an accelerated VPN connection and is enabled by default.

    Rules and Restrictions

  • Accelerated VPN connections are supported only for Site-to-Site VPN connections attached to a transit gateway (not virtual private gateways).
  • Certificate-based authentication for VPN connections may be incompatible with Global Accelerator due to limited support for packet fragmentation.
  • If using certificate-based authentication, the customer gateway device must support IKE fragmentation.

    Configuration Notes

  • The IKE negotiation for accelerated VPN tunnels must be initiated from the customer gateway device.
  • The two tunnel options that affect this behavior are Startup Action and DPD Timeout Action (Dead Peer Detection).
  • Ensure the NAT-T setting is correct in your configuration files (this can be downloaded from the VPC console).
