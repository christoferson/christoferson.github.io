AWS Transit Gateway Connect - Introduction and Overview

Core Problem and Solution

  • Traditional Challenge:
    • SD-WANs need to extend to cloud
    • Traditional infrastructure not well-suited
    • Complex IPsec VPN configurations required
    • High operational burden
  • Solution: Transit Gateway Connect:
    • Native SD-WAN integration with AWS
    • Uses GRE instead of IPsec
    • Supports dynamic routing (BGP)
    • Eliminates static route configuration

Key Concepts

  • New Components:
    • Connect Attachment:
      • New TGW attachment type
      • Uses existing VPC or Direct Connect as transport
      • Implements GRE and BGP protocols
    • Connect VPC:
      • Central VPC hosting third-party appliances
      • EC2 instances running virtual appliances
      • Links to TGW via Connect attachment
    • Transport Attachment:
      • Existing VPC or Direct Connect attachment
      • Serves as underlying transport

Exam Tips

  • Key Differentiators:
    • GRE replaces IPsec VPN
    • Dynamic routing with BGP required
    • Higher bandwidth (up to 20 Gbps)
    • Native integration with Network Manager
  • Remember:
    • Connect attachment requires transport attachment
    • Third-party appliances run in Connect VPC
    • BGP is mandatory (no static routes)
    • Supports both VPC and Direct Connect as transport

AWS Transit Gateway Connect - Architecture and Implementation

Architecture Models

  • Virtual Appliance Model:
    • Third-party appliances run in Connect VPC
    • Uses VPC attachments as transport
    • GRE tunnels between appliance and Transit Gateway
    • BGP for dynamic routing
  • Direct Connect Model:
    • Branch/customer gateway appliances on-premises
    • Uses Direct Connect as transport
    • BGP peering over GRE tunnel
    • Encapsulation channel for traffic exchange

Implementation Steps

  • Transit Gateway Configuration:
    • 1. Allocate CIDR block for Transit Gateway
    • 2. Create/modify Transport attachment (VPC type)
    • 3. Create Connect attachment using Transport
    • 4. Configure Connect peer (GRE tunnel)
  • GRE and BGP Setup:
    • Specify GRE parameters
    • Configure BGP parameters:
      • Support for iBGP
      • eBGP requires multi-hop configuration
    • Create appropriate route table entries

High Availability Design

  • Recommended Configuration:
    • Deploy appliances across multiple AZs
    • Create multiple Connect Peers
    • Configure redundant GRE tunnels
    • Implement BGP for failover
  • Verification Steps:
    • Test GRE tunnel connectivity
    • Verify BGP peer reachability
    • Confirm route exchange
    • Validate failover scenarios

Technical Limitations

  • Key Constraints:
    • Maximum 4 Connect peers per attachment
    • 5 Gbps per GRE tunnel limit
    • BGP required (no static routes)
    • GRE outer IP doesn't respond to ICMP
  • Support Features:
    • IPv6 support
    • MP-BGP for route advertisements
    • Multiple tunnel load balancing
    • Hourly attachment charges apply

Exam Tips

  • Architecture Focus:
    • Understand both deployment models:
      • Virtual appliance in VPC
      • Direct Connect with on-premises
    • Know implementation steps order
    • Remember HA design principles
  • Technical Requirements:
    • BGP is mandatory
    • GRE tunnel configuration
    • Transport attachment prerequisites
    • Maximum bandwidth limitations
  • Common Scenarios:
    • High availability setup
    • Bandwidth scaling
    • Route advertisement
    • Failover configuration
