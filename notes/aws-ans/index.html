Networking Services

  • Network Foundations: Core networking infrastructure services
    • Amazon VPC: Virtual Private Cloud
      • Isolated virtual network in AWS cloud
      • Define IP ranges, subnets, routing tables
      • Control network security and connectivity
    • Transit Gateway: Network Transit Hub
      • Centrally manage network connections
      • Hub-and-spoke connectivity model
      • Connect VPCs and on-premises networks
    • PrivateLink: Private Service Access
      • Private connectivity to AWS services
      • Avoid public internet exposure
      • Service endpoint integration
  • Hybrid Connectivity: On-premises to cloud integration services
    • Direct Connect: Dedicated Network Connection
      • Private dedicated network connection to AWS
      • Consistent network performance
      • Reduced data transfer costs
    • Cloud WAN: Wide Area Network Service
      • Unified global network management
      • Connect on-premises, branch offices, and cloud
      • Centralized policy control
    • Client VPN: Remote Access VPN
      • Secure remote access to AWS and on-premises
      • OpenVPN-based solution
      • Managed client endpoints
    • Site-to-Site VPN: IPsec VPN Connection
      • Encrypted connection between VPC and on-premises
      • Support for static and dynamic routing
      • High availability options
  • Edge Networking: Global content delivery and traffic management services
    • CloudFront: Content Delivery Network (CDN)
      • Global content delivery with low latency
      • Edge caching for improved performance
      • Integration with WAF and Shield for security
    • Amazon Route 53: DNS Service
      • Highly available and scalable DNS service
      • Advanced routing policies (latency, geolocation, weighted)
      • Health checking and DNS failover
    • Global Accelerator: Network Performance Service
      • Improve global application availability and performance
      • Static IP addresses for global routing
      • Automatic failover and traffic distribution
  • Application Networking: Application-level networking and service discovery
    • API Gateway: API Management Service
      • Create, publish, and manage APIs at scale
      • Request/response transformation and validation
      • API versioning and stage management
    • App Mesh: Service Mesh Technology
      • Application-level networking for microservices
      • End-to-end visibility and traffic control
      • Service-to-service communication management
    • Cloud Map: Service Discovery
      • Service registration and discovery
      • Real-time health monitoring
      • DNS and API-based service lookup
  • Network Security: Protection and security control services
    • Firewall Manager: Centralized Security Management
      • Central management of security rules
      • Multi-account security policy administration
      • Automated security group management
    • Shield: DDoS Protection
      • DDoS attack protection (Standard and Advanced)
      • Always-on network flow monitoring
      • Integration with other AWS services
    • Network Firewall: Network Traffic Control
      • VPC-level network protection
      • Stateful and stateless filtering
      • Traffic inspection and control
    • WAF: Web Application Firewall
      • Protect web applications from common exploits
      • Custom rule creation and management
      • Integration with CloudFront and ALB

Public IP Address

This is a large range of IP addresses. But a year after IPv4 was introduced, the Internet Architecture Board (IAB) and supporting community realized there were not enough addresses as more and more devices were connected to the internet. Every laptop, mobile phone, tablet, server, Internet of Things (IoT) device and so on needs at least one IP address to connect to the global network, the internet.

The IPv4 range is split using classful addressing into smaller ranges.

  • Class A: (Primary Range)
    • Range: 0.0.0.0 - 127.255.255.255
    • Capacity: Over 2.1 billion IP addresses
  • Class B: (Secondary Range)
    • Range: 128.0.0.0 - 191.255.255.255
    • Capacity: Over 1 billion IP addresses
  • Class C: (Tertiary Range)
    • Range: 192.0.0.0 - 223.255.255.255
    • Capacity: Over 2 million IP addresses
  • Additional Classes:
    • Class D and Class E ranges also exist

Note: All of these IP addresses are publicly routed to connect to these using public IP addresses.

Private IP Address

There were also ranges created for private networks. These IP addresses cannot communicate over the internet and they are private. The private IPv4 addresses were also broken down into ranges.

  • Private Class A: (Large Private Networks)
    • Range: 10.0.0.0 - 10.255.255.255
    • Capacity: One single Class A network
    • Size: 16,777,216 addresses
  • Private Class B: (Medium Private Networks)
    • Range: 172.16.0.0 - 172.31.255.255
    • Capacity: 16 Class B networks
    • Size: 1,048,576 addresses
  • Private Class C: (Small Private Networks)
    • Range: 192.168.0.0 - 192.168.255.255
    • Capacity: 256 Class C networks
    • Size: 65,536 addresses

AWS Note: The default Amazon VPC in AWS is configured using a Class B range.

Important: For private IP addresses to communicate with the internet, they have to use network address translation (NAT).
