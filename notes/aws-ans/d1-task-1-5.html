AWS Certified Advanced Networking - Specialty (ANS-C01) / Domain 1 Task 1-5

Domain 1: Network Design

Task Statement 1.5: Design a routing strategy and connectivity architecture between on-premises networks and the AWS Cloud

Knowledge 1: Routing Fundamentals

Understanding core routing concepts and protocols:

  • Static Routing:
    • Manual route configuration
    • Simple and predictable paths
    • Low overhead
    • Best for simple networks
  • Dynamic Routing:
    • BGP (Border Gateway Protocol):
      • Path selection and propagation
      • AS path attributes
      • Route aggregation
      • Policy-based routing
    • Route advertisement and learning
    • Automatic failover capabilities
    • Scalability benefits

Knowledge 2: Layer 1 and Layer 2 Concepts

  • Physical Interconnects:
    • VLAN (Virtual LAN):
      • Network segmentation
      • Traffic isolation
      • Broadcast domain control
    • LAG (Link Aggregation Group):
      • Bandwidth aggregation
      • High availability
      • Load balancing
    • Optics:
      • Fiber types and specifications
      • Wavelength considerations
      • Power levels and loss budgets
    • Jumbo Frames:
      • MTU size considerations
      • Performance optimization
      • Compatibility requirements

Knowledge 3: Encapsulation and Encryption Technologies

  • GRE (Generic Routing Encapsulation):
    • Tunnel configuration
    • Protocol encapsulation
    • Multiprotocol support
  • IPsec:
    • Security protocols
    • Authentication methods
    • Encryption algorithms
    • Key management

Knowledge 4: Resource Sharing and Overlay Networks

  • AWS Account Resource Sharing:
    • RAM (Resource Access Manager)
    • Cross-account permissions
    • Resource sharing policies
  • Overlay Networks:
    • Virtual network architecture
    • Network virtualization
    • VXLAN and NVGRE

Essential Skills for Implementation

  • Hybrid Connectivity Requirements:
    • Bandwidth assessment
    • Latency requirements
    • High availability needs
    • Security compliance
  • Redundant Connectivity Design:
    • Direct Connect configuration
    • VPN backup
    • Multi-region considerations
  • BGP Design:
    • AS path manipulation
    • Route preference
    • Load sharing configuration
  • SD-WAN Integration:
    • Transit Gateway Connect
    • Overlay network design
    • Traffic optimization

Best Practices and Design Considerations:

  • Implement redundant connections for high availability
  • Use BGP for dynamic routing when possible
  • Consider bandwidth requirements and scalability
  • Implement proper security measures
  • Monitor network performance and health
  • Plan for disaster recovery
  • Document network architecture and configurations

Common Design Patterns:

  • Direct Connect with VPN backup
  • Multi-region connectivity
  • Hub-and-spoke network architecture
  • SD-WAN integration
  • Hybrid DNS architecture
