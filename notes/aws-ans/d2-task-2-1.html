Knowledge 1: Routing Protocols and VPNs

  • Routing Protocols (Network Path Control):
    • Static Routing (Manual Path Configuration):
      • Manual Route Configuration: Administrator-defined network paths
        • Route Components:
          • Destination CIDR block
          • Next hop target
          • Route priority (metric)
        • Use Cases:
          • Simple network topologies
          • Predictable traffic patterns
          • Security-controlled paths
        • Limitations:
          • No automatic failover
          • Manual maintenance required
          • Scale limitations
      • Default Routes: Catch-all routing configuration
        • Implementation:
          • 0.0.0.0/0 route configuration
          • Last resort path definition
          • Internet gateway routing
        • Best Practices:
          • Careful placement in route tables
          • Security considerations
          • Failover planning
    • Dynamic Routing (Automated Path Learning):
      • BGP Configuration: Industry-standard exterior gateway protocol that enables networks to exchange routing information and make path decisions based on multiple criteria
        • Essential Components:
          • ASN assignment: Unique identifier (16-bit or 32-bit) that represents autonomous systems in BGP routing
            • Public ASN (assigned by IANA): Used for internet routing
            • Private ASN (64512-65534): Used for internal routing
          • Neighbor relationships: BGP peers that exchange routing information using TCP port 179
            • EBGP: External BGP between different ASNs
            • IBGP: Internal BGP within same ASN
          • Route advertisements: Process of announcing network prefixes to BGP peers
            • Prefix announcements
            • Path attributes
            • Route filtering
        • Path Attributes: Criteria used by BGP to select best path to destination
          • AS_PATH: Sequence of ASNs that describes path to destination
          • LOCAL_PREF: Higher value preferred, influences outbound traffic
          • MED: Lower value preferred, influences inbound traffic from neighbor AS
      • MACsec (IEEE 802.1AE MAC Security Standard):
        • Overview: Layer 2 security protocol that provides point-to-point encryption, integrity, and authenticity between network devices
          • Security Features:
            • Data encryption: AES-GCM encryption of all Layer 2 traffic
            • Integrity checking: Per-packet authentication and integrity verification
            • Replay protection: Prevention of packet replay attacks
          • AWS Implementation:
            • Direct Connect support: Available on dedicated connections 10Gbps or higher
            • Key management: Automatic key rotation every 24 hours
            • Connectivity requirements: Both ends must support MACsec
          • Use Cases:
            • Regulatory compliance: Meeting data-in-transit encryption requirements
            • Security enhancement: Additional layer of encryption for sensitive workloads
            • Physical security: Protection against wire-tapping and man-in-the-middle attacks
  • VPN Solutions (Secure Connectivity):
    • Site-to-Site VPN: Secure tunnel between networks
      • IPSec Configuration: Suite of protocols providing secure communication over IP networks through authentication and encryption
        • Phase 1 (IKE): Initial exchange to establish secure channel for Phase 2 negotiation
          • Authentication methods: Ways to verify identity of VPN peers
            • Pre-shared keys: Symmetric key known to both parties
            • Certificate-based: Using digital certificates for authentication
          • Encryption algorithms: Methods to secure IKE negotiation
            • AES-128: Standard encryption for good security
            • AES-256: Stronger encryption for higher security requirements
          • DH groups: Define strength of key exchange process
            • Group 2 (1024-bit): Legacy support
            • Group 14-24: Stronger security options
      • Tunnel Options:
        • Configuration Parameters:
          • Inside tunnel CIDR
          • Outside tunnel IP
          • Routing options (static/dynamic)
        • Advanced Features:
          • Dead peer detection
          • Tunnel monitoring
          • MTU configuration
    • Accelerated VPN: Enhanced performance VPN
      • Global Accelerator Integration:
        • Benefits:
          • Reduced latency
          • Improved availability
          • Consistent performance
        • Implementation:
          • Endpoint configuration
          • Health monitoring
          • Traffic distribution

Knowledge 2: Physical Connectivity

  • Layer 1 Hardware (Physical Network Infrastructure):
    • Direct Connect Requirements (Dedicated Network Connection to AWS):
      • LOA (Letter of Authorization): Official document from AWS authorizing cross-connect installation
        • Components:
          • Port assignments: Specific port information for cross-connect
          • Facility details: Location and cage information
          • Circuit ID: Unique identifier for the connection
        • Process:
          • AWS generates LOA-CFA
          • Valid for 90 days
          • Required by colocation provider
      • Colocation Facilities: Physical locations where Direct Connect connections are established
        • Types:
          • AWS Direct Connect locations: AWS-approved facilities
          • Partner locations: Third-party provider facilities
          • Hosted connections: Partner-managed connections
        • Requirements:
          • Meet AWS specifications
          • Support required bandwidth
          • Provide necessary redundancy
      • Cross-connect Specifications: Physical cable requirements for DX connection
        • Cable Types:
          • Single-mode fiber: Required for all DX connections
          • 1000BASE-LX: For 1Gbps connections
          • 10GBASE-LR: For 10Gbps connections
          • 100GBASE-LR4: For 100Gbps connections
        • Specifications:
          • Maximum distance: Typically up to 10km
          • Connector type: LC-LC for most connections
          • Wavelength requirements: 1310nm
    • Physical Interface Types (Connection Standards):
      • 1 Gigabit Ethernet:
        • 1000BASE-LX: Single-mode fiber interface for 1Gbps
        • Auto-negotiation: Disabled by default
        • Full-duplex only
      • 10 Gigabit Ethernet:
        • 10GBASE-LR: Single-mode fiber interface for 10Gbps
        • No auto-negotiation
        • Full-duplex operation
      • 100 Gigabit Ethernet:
        • 100GBASE-LR4: Single-mode fiber interface for 100Gbps
        • MACsec support: Available for encryption
        • Enhanced reliability features
    • Bandwidth Options (Connection Capacities):
      • Dedicated Connections:
        • 1 Gbps: Entry-level dedicated connection
        • 10 Gbps: Standard high-bandwidth option
        • 100 Gbps: Maximum bandwidth option
      • Hosted Connections:
        • 50 Mbps to 10 Gbps: Flexible bandwidth options
        • Scalable on demand
        • Partner-provided options
  • Layer 2 and 3 Components (Network Protocol Implementation):
    • VLANs (Virtual Local Area Networks - Logical network segmentation):
      • VLAN Tagging: IEEE 802.1Q frame identification method for separating traffic
        • Components:
          • VLAN ID (VID): 1-4094 range, unique identifier for each VLAN
          • Tag Protocol ID (TPID): Set to 0x8100 for 802.1Q
          • Priority Code Point (PCP): QoS marking (0-7)
        • AWS Implementation:
          • Virtual Interfaces (VIFs): Direct Connect logical interfaces using VLAN tags
            • Private VIF: Connect to VPC using private ASN
            • Public VIF: Connect to AWS public services
            • Transit VIF: Connect to Transit Gateway
          • VLAN Limitations:
            • One VLAN per VIF
            • Unique VLAN IDs required
            • Cannot reuse VLANs across connections
      • Trunking: Method for carrying multiple VLANs over a single physical link
        • Configuration Requirements:
          • Native VLAN: Untagged VLAN on trunk (typically avoided in AWS)
          • Allowed VLANs: Explicit VLAN permission on trunk
          • Encapsulation: 802.1Q required for AWS
        • AWS Considerations:
          • LAG support: Link Aggregation for increased bandwidth
          • Maximum MTU: 9001 bytes (Jumbo frames)
          • Trunk negotiation: Manual configuration required
      • QoS Configuration: Traffic prioritization and bandwidth management
        • Classification Methods:
          • 802.1p priority: Layer 2 QoS marking (0-7)
          • DSCP: Layer 3 QoS marking (0-63)
          • Traffic class mapping: QoS policy implementation
        • AWS QoS Implementation:
          • Direct Connect priority levels: Strict priority queuing
          • Bandwidth allocation: Per-queue minimum guarantees
          • Traffic shaping: Rate limiting capabilities
    • IP Addressing (Network Layer Addressing Strategy):
      • Subnet Design: Logical division of IP networks
        • Design Principles:
          • Hierarchical addressing: Structured IP allocation
          • Zone alignment: AZ-based subnet distribution
          • Function-based segregation: Purpose-specific subnets
        • AWS Implementation:
          • VPC subnets: Region-specific network segments
          • Reserved addresses: First 4 and last IP in each subnet
          • Subnet sizing: /16 to /28 range supported
      • CIDR Planning: IP address range allocation strategy
        • VPC CIDR Requirements:
          • Primary CIDR: Main IP range for VPC
            • Size: /16 to /28
            • RFC 1918 ranges recommended
            • Non-overlapping requirement
          • Secondary CIDRs: Additional IP ranges
            • Maximum 5 ranges
            • Must not overlap
            • Contiguous ranges allowed
      • Address Translation: IP address mapping mechanisms
        • NAT Types:
          • NAT Gateway: Managed NAT service
            • Source NAT capability
            • AZ-specific deployment
            • Automatic scaling
          • NAT Instance: Self-managed NAT solution
            • Custom NAT rules
            • Port forwarding support
            • Manual scaling required
    • Routing and Switching (Path Determination and Forwarding):
      • Gateway Configuration: Network entry and exit points
        • Gateway Types:
          • Internet Gateway: Public internet connectivity
          • Virtual Private Gateway: VPN and Direct Connect termination
          • Transit Gateway: Hub-and-spoke connectivity
        • Configuration Elements:
          • Attachment options
          • Route propagation settings
          • Security policies
