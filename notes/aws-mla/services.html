Services - Analytics

Amazon Athena

Amazon Athena is an interactive query service that makes it easy to analyze data directly in Amazon S3 using standard SQL. It's serverless, so there's no infrastructure to set up or manage.

Key Features:

  • Serverless: No need to provision or manage servers
  • Pay-per-query: You only pay for the queries you run
  • Supports various data formats: CSV, JSON, ORC, Avro, and Parquet
  • Integrates with AWS Glue Data Catalog
  • JDBC/ODBC driver support for connecting with business intelligence tools

Exam-specific information:

  • Data Preparation: Athena is crucial for querying and transforming data stored in S3, which is a common task in ML workflows
  • Cost-effective: Understanding when to use Athena vs. other query services is important for optimizing costs
  • Integration: Know how Athena integrates with other AWS services like Glue and QuickSight
  • Performance optimization: Understand how data format and partitioning affect query performance

Amazon Kinesis Data Firehose

Amazon Kinesis Data Firehose is a fully managed service for delivering real-time streaming data to destinations such as Amazon S3, Amazon Redshift, Amazon Elasticsearch Service, and Splunk.

Key Features:

  • Automatic scaling: Handles any amount of streaming data and scales automatically
  • Near real-time: Delivers data to destinations within 60 seconds
  • Supports data transformation: Can transform data using Lambda functions
  • Supports various data formats and compression types
  • Integrates with Kinesis Data Streams and Kinesis Data Analytics

Exam-specific information:

  • Data Ingestion: Understand how Firehose can be used to ingest streaming data for ML workflows
  • Data Transformation: Know how to use Lambda with Firehose for data cleaning and feature engineering
  • Integration: Understand how Firehose works with other Kinesis services and ML-related AWS services
  • Use cases: Be familiar with scenarios where Firehose is the appropriate choice for data ingestion

Amazon EMR (Elastic MapReduce)

Amazon EMR is a cloud-native big data platform for processing vast amounts of data using open source tools such as Apache Spark, Apache Hive, Apache HBase, Apache Flink, Apache Hudi, and Presto.

Key Features:

  • Managed cluster platform: Simplifies running big data frameworks
  • Flexible: Supports various instance types and can use Spot Instances
  • Scalable: Can easily add and remove instances from a cluster
  • Supports various storage options: HDFS, EMRFS (S3), and local file system
  • Integrates with other AWS services like S3, DynamoDB, and Redshift

Exam-specific information:

  • Data Processing: Understand how EMR can be used for large-scale data processing and feature engineering
  • Machine Learning: Know how to use EMR for distributed machine learning with frameworks like Spark MLlib
  • Cost Optimization: Understand strategies for optimizing EMR costs, such as using Spot Instances
  • Use Cases: Be familiar with scenarios where EMR is the appropriate choice over other data processing services
  • Integration: Know how EMR integrates with other AWS services in ML workflows

Key Features:

  • Job Orchestration: Supports scheduling and dependency resolution for ETL jobs

  • Support for various data sources and targets: Works with relational databases, data warehouses, and data lakes

Exam-specific information:

  • ETL Job Creation: Be familiar with creating and managing ETL jobs using Glue's visual or code-based interfaces
