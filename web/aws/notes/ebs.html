AWS EBS Notes

Volume Types

  1. Solid state drives (SSD) transactional workloads
    1. General Purpose SSD — Provides a balance of price and performance. (gp3 and gp2)
      (gp3) 3,000 IOPS and 125 MB/s of consistent performance
    2. Provisioned IOPS SSD — Provides high performance for mission-critical, low-latency, or high-throughput workloads. (io2 Block Express, io2, and io1)
      single digit millisecond latencies
  2. Hard disk drives (HDD) throughput workloads
    1. Throughput Optimized HDD — A low-cost HDD designed for frequently accessed, throughput-intensive workloads. (st1)
    2. Cold HDD — The lowest-cost HDD design for less frequently accessed workloads. (sc1)
  1. modify the capacity, performance, or type of an existing EBS volume on the fly

Encrypted Volumes

  1. Data at rest inside the volume is encrypted
  2. Any snapshot created from the volume is encrypted
  3. Data moving between the volume and the instance is encrypted
https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/ebs-fast-snapshot-restore.html Amazon EBS fast snapshot restore (FSR) enables you to create a volume from a snapshot that is fully initialized at creation. This eliminates the latency of I/O operations on a block when it is accessed for the first time. Volumes that are created using fast snapshot restore instantly deliver all of their provisioned performance.
