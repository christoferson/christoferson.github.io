AWS Route 53 Notes

  1. Domain Name System (DNS) - Internet Service that converts domain names into routable IP addresses
  2. IPv4 - Internet Protocol v4 - 32-Bit address space
  3. IPv6 - Internet Protocol v6 - 128-Bit address space
  4. Top Level Domain - e.g. 'com' in acme.com
  5. Second Level Domain - e.g. 'co' in acme.co.uk
  6. Domain Registrar - 3rd Party Company where you register your domain
  7. Name Server - Server which contains the DNS records
  8. Start of Authority (SOA) - Provide information about the domain
  9. Time-ti-Live (TTL - length of time that a DNS record gets cached
  10. Route53 can point to ELB, EC2, API gateway, CloudFront, EIP
  11. Alias Option for A Record - Allows you to select AWS resource as target instead of specifying IP
  12. Traffic Flow - Visial Editor that lets you create sophisticated routing configurations

Record Types

  1. Name Server Records (NS) - Use by top-level domain servers to direct traffic to the DNS server containing the authoritative DNS records.
  2. Address Records (A Record) - Convert the name of a domain into an IP Address
  3. Canonical Names (CNAME) - Resolve one domain name to another - rather than an IP address

Routing Policies

  1. Simple – Map one record to one or more IP. IP are returned in random order.
  2. Failover – Use when you want to configure active-passive failover. Define a primary and secondary record. Route53 will failover to secondary if health check fails.
  3. Geolocation – Use when you want to route traffic based on the location where the request originated from.
  4. Geoproximity – Use when you want to route traffic based on the location of your resources and, optionally, shift traffic from resources in one location to resources in another. Requires Traffic Flow to specify Bias.
  5. Latency – Create record for each region, AWS will direct to the record/region with the best latency with less round-trip time.
  6. Multivalue answer – Use when you want Route 53 to respond to DNS queries with up to eight healthy records selected at random. Simple with health checks.
  7. Weighted – Create multiple record with different weights and target. e.g. 85% to ALB, 15% to ALB New

Health Checks

  1. Check health every 30s by default. Min 10s.
  2. Can trigger CloudWatch alarm.
  3. Health Checks can chain.
  4. Support both AWS and Non-AWS Endpoints

Route53 Resolver .2 Resolver

  1. Regional Service that lets you route DNS queries between VPC and on-premise. Hybrid Environments.

Private Hosted Zone

  1. Need EnableDNSHostName and EnableDNSSupport set to true
