AWS Identity and Access Management Notes

  1. Root User - Single sign-in identity that has complete access to all AWS services and resources in the account
  2. Policies - JSON document that define permissions for specific user, group, or role
  3. IAM Credentials Report (account-level)
  4. IAM Access Advisor (user-level)

IAM Identities

  1. Users - End Users
  2. Groups - Grouping of Users
  3. Roles - an IAM entity that defines a set of permissions for making AWS service requests. IAM roles are not associated with a specific user or group. Instead, trusted entities assume roles, such as IAM users, applications, or AWS services such as EC2.

Policy Types

  1. Identity-based - permissions policies that you attach to an IAM identity, such as an IAM user, group, or role.
    1. Managed policies – Standalone identity-based policies that you can attach to multiple users, groups, and roles in your AWS account.
      1. AWS managed policies – Managed policies that are created and managed by AWS.
      2. Customer managed policies – Managed policies that you create and manage in your AWS account.
    2. Inline policies – Policies that you create and manage and that are embedded directly into a single user, group, or role.
  2. Resource-based - permissions policies that you attach to a resource such as an Amazon S3 bucket or an IAM role trust policy.
