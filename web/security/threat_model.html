Threat Grammar Study Guide

Overview: This study guide covers the structured approach to writing consistent and concise threat statements using threat grammar methodology.

1. Basic Threat Grammar Structure

Formula:

[threat source] [prerequisites] can [threat action], which leads to [threat impact], negatively impacting [impacted assets]

Components Breakdown:

Threat Source

Definition: The entity taking action

Examples:

  • A threat actor (useful default)
  • An internet-based threat actor
  • An internal or external actor

Prerequisites

Definition: Conditions or requirements that must be met for the threat to be viable

Examples:

  • With access to another user's token
  • Who has administrator access
  • With user permissions
Note: If no prerequisites exist, consider decomposing the threat

Threat Action

Definition: The action being performed by the threat source

Examples:

  • Spoof another user
  • Tamper with data stored in the database
  • Make thousands of concurrent requests

Threat Impact

Definition: The direct impact of a successful threat action

Examples:

  • Unauthorized access to user's bank account information
  • Modifying the username for all-time high score
  • Web application unable to handle other user requests

Impacted Assets

Definition: The assets affected by a successful threat action

Examples:

  • User banking data
  • Video game high score list
  • The web application

Sample Basic Statements:

1. An internet-based threat actor with access to another user's token can spoof another user, which leads to viewing the user's bank account information, negatively impacting user banking data.
2. An internal threat actor who has administrator access can tamper with data stored in the database, which leads to modifying the username for the all-time high score, negatively impacting the video game high score list.

2. Advanced Threat Grammar Structure

Enhanced Formula:

[threat source] [prerequisite] can [threat action], which leads to [threat impact], resulting in reduced [impacted goal] of [impacted asset]

Additional Component:

Impacted Goal

Definition: The information security or business objective that is negatively affected

Common Examples (CIA Triad):

  • Confidentiality
  • Integrity
  • Availability

Sample Advanced Statement:

A threat actor with user permissions can make thousands of concurrent requests, which leads to blocking user access to the application, resulting in reduced availability of the web application.

3. Key Guidance Principles

Purpose of Each Element:

Input Elements (help identify mitigations):

  • [threat source]
  • [prerequisites]
  • [threat action]

Impact Elements (help prioritize threats):

  • [threat impact]
  • [impacted goal]
  • [impacted assets]

Flexibility Rules:

  • Each field is optional as long as the statement makes sense
  • Write down elements that come to mind initially
  • Fill in more elements in subsequent passes
  • Consider decomposition for more specificity

4. Decomposition Strategy

When to Decompose:

  • When you don't know all threat components initially
  • When a single threat can be broken into more specific sub-threats

Example Process:

Initial Broad Statement:

Information is disclosed unintentionally from the S3 bucket, resulting in reduced confidentiality of user vehicle registration documents.

Decomposed into Specific Threats:

1. A threat actor with access to inspect traffic between the user and S3 endpoint can view data-in-transit, resulting in reduced confidentiality of user vehicle registration documents.
2. An internal actor can access the data stored in S3, resulting in reduced confidentiality of user vehicle registration documents.

5. Threat Identification Strategies

For Frequent Threat Types:

  • Pivot your approach
  • Think about all elements the threat applies to
  • Document comprehensively

Alternative Approaches:

  • Apply threats to zones of trust instead of specific elements
  • Use wildcard (*) placeholders for uncertain areas
  • Remove duplicate entries after comprehensive documentation

6. Implementation Tips

Common Challenges:

  • Remembering the grammar structure
  • Understanding the purpose of each field
  • Knowing when to decompose threats

Solutions:

  • Use web-based tools that follow threat grammar
  • Leverage suggestion features
  • Practice with the structured approach regularly
